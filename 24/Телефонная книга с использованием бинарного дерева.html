Задание: Телефонная книга с использованием бинарного дерева

Описание задачи:

Ваша задача - разработать программу для хранения и поиска контактов в телефонной книге. Оптимальным решением для этой задачи будет использование бинарного дерева, которое позволяет эффективно добавлять, искать и отображать контакты.

Требования к реализации:

  1. Создайте два класса: Contact и PhoneBook.

  2. Класс Contact представляет отдельный контакт. У него должны быть следующие атрибуты:

    • name (str): Имя контакта.
    • phone_number (str): Номер телефона контакта.
    • left (Contact): Левый потомок (контакт с алфавитно меньшим именем).
    • right (Contact): Правый потомок (контакт с алфавитно большим именем).

  3. Класс PhoneBook представляет собой саму телефонную книгу. У него должны быть следующие методы:

    • add_contact(name: str, phone_number: str) -> None: Добавляет новый контакт в телефонную книгу. Если контакт с таким именем уже существует, обновляет его номер телефона.
    • search_contact(name: str) -> Contact: Ищет контакт по имени и возвращает его объект. Если контакт не найден, возвращает None.
    • display_contacts() -> None: Выводит все контакты в телефонной книге в отсортированном порядке по имени.

Пример использования:

# Создаем телефонную книгу
phone_book = PhoneBook()

# Добавляем контакты
phone_book.add_contact("John Doe", "+123456789")
phone_book.add_contact("Jane Smith", "+987654321")
phone_book.add_contact("Alice Williams", "+111111111")

# Ищем контакт
result = phone_book.search_contact("Jane Smith")
if result:
    print(f"Contact found - Name: {result.name}, Phone: {result.phone_number}")
else:
    print("Contact not found")

# Выводим все контакты
phone_book.display_contacts()

Примечание:

  • Проверьте, что ваша реализация корректно обрабатывает ситуации, когда контакт с таким именем уже существует в телефонной книге.

  • Убедитесь, что методы правильно обходят и выводят все контакты в телефонной книге в отсортированном порядке по имени.

  • Добавьте подробные комментарии к вашему коду для облегчения его понимания другими разработчиками.
